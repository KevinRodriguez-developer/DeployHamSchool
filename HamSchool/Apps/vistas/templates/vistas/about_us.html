About Ham-School

The best english option

This project consists of teaching interactive English courses to primary and secondary school children through a digital medium, with the purpose of helping their learning and orienting them to better learning.

About the project

Objective

Get to position ourselves as a website that is used in schools to improve student learning in a period of 5 to 6 years.

Public Objective

The creation of HAM-SCHOOL is a website that is aimed at preschool-secondary children, with the aim that children learn or reinforce the English language since not all schools teach English.

Usability

With the HAM-SCHOOL website we offer a solution to these problems by creating a website that is capable of teaching an interactive course which draws the child's attention with various games that are capable of teaching English topics in a simple way, based on the help from a professional in the field of English and with this teach the specific topics that are required in the children's age.

Arquitecture

- Model: manipulation of application data

- Template: decide how the data will be displayed in the browser

- View: decide which data the templates will show

Design Patterns - Singleton Pattern

The singleton pattern is one of the simplest design patterns: it involves only one class which is responsible to instantiate itself, to make sure it creates not more than one instance; in the same time it provides a global point of access to that instance. In this case the same instance can be used from everywhere, being impossible to invoke directly the constructor each time.

- Ensure that only one instance of a class is created.

- Provide a global point of access to the object.

Requirements Functional

- Ham-School is a web app that will provide knowledge for learning at the level of each user.

- The web app will have periodical updates to keep the website attractive to users.

- The web app will save user data to have better monitoring of each user.

Requirements not functional

- The web app should have an user-friendly interface.

- The web app should be constant maintenance.

- A database will save information entered in the app and the information in general.
